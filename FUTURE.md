# 逸承工作室未來發展

最後更新：2026-04-07

## 目前已確立方向

- 品牌名稱：逸承工作室
- 主風格：未來感為主，遊戲感為輔
- 主要方法：AI Coding
- 主案例：Warmth House
- 現階段部署：GitHub Pages
- 長期方向：科技新知、遊戲內容、3C 商城與平台型網站

## 核心守則

- 前台不能寫成說明頁或提案頁
- AI 備註、規劃與提示必須留在 Markdown，不可直接出現在網站畫面
- 單頁文字量要克制，避免滿版長文
- 圖文面積比以 `1.5:1` 為目標，視覺模組要大於純文字
- 整體主風格以未來感為主，遊戲感為輔

詳細規則請以 `RULE.md` 為準。

## 現況重新評估

目前網站已經不是單純雛形，而是一個有明確視覺方向的互動式首頁原型。

目前最強的地方：

- Hero 的辨識度高
- 掃描器控制台有記憶點
- 互動感已經成立
- 網站已正式上線可分享

目前最弱的地方：

- 後段內容強度跟不上 Hero
- 商業轉換能力不足
- 案例體系還不完整
- 聯絡入口還不是真正可承接詢問的入口
- 仍偏向概念型首頁，而不是完整官網

## 目前網站定位

現階段這個網站比較接近：

- 有品牌語言的工作室首頁
- 互動式展示頁
- 可延伸為完整官網的基底

但還不是：

- 完整可成交的工作室網站
- 完整內容型科技平台
- 具商城能力的正式產品站

## 下一階段重點

### 第一階段：從概念首頁變成能接案的官網

這是目前最優先的事情。

要補上的重點：

- 真實聯絡方式
- 更完整的主案例頁
- 更清楚的服務頁
- 統一品牌語言
- 補齊基礎收尾，例如 favicon、SEO、社群預覽圖
- 逐步把示例動畫替換成自己的視覺語言

### 第二階段：從單頁延伸到內容型架構

等第一階段穩定後，網站應該補成多頁結構：

- `Home`
- `Services`
- `Case Study`
- `About`
- `Contact`

之後再往內容分類擴張：

- `Tech`
- `Game`
- `3C`

### 第三階段：平台化與商業模組

未來如果網站要往像三創那樣的方向走，應先累積內容與入口，再決定商城深度。

先做：

- 科技內容入口
- 遊戲內容區
- 商品卡與導購頁

再決定：

- 是否要做完整商城
- 是否要導入會員、購物車、搜尋與分類系統

## 我建議的技術演進

### 現在

維持：

- 靜態 HTML
- GitHub Pages
- CDN 型互動效果

理由：

- 成本低
- 改動快
- 很適合現在快速確立風格

### 下一步

當頁面和內容變多後，建議考慮改用 `Astro`。

原因：

- 適合靜態內容網站
- 很適合案例頁、文章頁、分類頁
- 元件化後更容易維護
- 對 GitHub Pages 友善

### 更後面

如果未來真的做商城或大型互動功能，再考慮更完整的框架：

- `Next.js`
- `Nuxt`
- 或外接現成電商系統

## 最合理的執行順序

1. 先做正式 `Case Study` 頁
2. 再做正式 `Services` 頁
3. 補真正的 `Contact` 入口
4. 補 `About`
5. 開始規劃內容分類頁
6. 等內容量起來後，再考慮 `Astro` 化

## 目前最值得優先加強的地方

### 1. Case Study

現在只有首頁案例區，還不夠。

應該做成完整案例頁，內容包括：

- 專案背景
- 問題
- 解法
- 頁面展示
- 上線成果

### 2. Services

目前服務區較像簡要介紹，之後需要變成真正可理解、可購買的服務頁。

### 3. Contact

目前仍偏 placeholder，需要換成真正的聯絡方式與 CTA。

### 4. 視覺一致性

Hero 已經很強，之後要讓 `Case`、`Services`、`Contact` 也擁有同樣級別的未來感語言。

## 結論

目前網站最適合的策略，不是繼續堆更多炫技效果，而是把 Hero 的完成度往整站延伸。

最關鍵的方向是：

- 先從互動首頁變成完整工作室官網
- 再從官網變成內容平台
- 最後再往科技平台與商城方向擴張
